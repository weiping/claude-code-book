# 序言：一个源码侦探的自白

Claude Code 的 npm 包里意外留了一个 60MB 的调试文件。1,902 个源文件、51 万行 TypeScript 代码，全部暴露。

第一次打开 `src/main.tsx` 时，文件长度是 8,247 行。我只是想搞清楚"这个 CLI 到底是怎么跑起来的"，但顺着 `launchRepl()` 往下追了三个文件之后，意识到自己掉进了一个与任何外部文档都不符的世界——这个工具的复杂度远超任何官方描述。

**这不是一个 CLI。这是一套 Agent Harness。**

「Harness」是工程术语，指把不可控的外部系统包裹在可控接口之内的框架结构。Claude Code 把 Anthropic 的模型变成一个可编排的 Agent 运行时——它的每一个子系统（Tool、Hook、Task、Swarm、Context Engineering）都在做同一件事：**在 AI 的不确定性和工程系统的确定性之间，修一道桥梁**。

写作本书过程中，最让我意外的不是某个具体实现，而是**演化的痕迹**。Hook 系统从 3 个事件增长到 27 个，Task 类型从 1 种扩展到 7 种，Swarm 从单进程演化为三种 Teammate 后端。每一次演化都留下了接口膨胀的痕迹——但没有一次是重写。这些「伤疤」恰恰是最有价值的工程证据：它们展示了真实系统如何在约束条件下渐进式扩展，而不是推倒重来。

---

## 这本书适合谁

**主要读者**：正在构建 AI Agent 系统的工程师和架构师。如果你在问「我的 Agent 系统应该用框架还是自己造？工具调用怎么设计才能可扩展？权限系统怎么避免成为瓶颈？」——本书提供了一个 50 万行生产代码的真实样本。

**次要读者**：对 AI 工程底层机制感兴趣、想理解现代 AI 应用基础设施的开发者。Claude Code 是目前罕见的、源码完整暴露的生产级 Agent Harness。

**不适合本书的读者**：只想了解 Claude Code 使用方法的用户。本书没有「如何使用 `/slash command`」或「如何配置 CLAUDE.md」的操作指南——这是一本拆解工程决策的技术书，不是操作手册。

## 如何阅读本书

全书分十篇，41 章正文，遵循两条并行线索：

**源码追踪线**：每章跟随一个具体的工程问题——「为什么 `QueryEngine.ts:436` 在进入 for-await 循环前先写入转录文件？」「为什么权限系统用三个函数而非一个？」——顺着源码追踪到设计决策的原因。

**模式提炼线**：每章末尾提炼一个可命名的工程模式，配有源码锚点和可携带性说明。第 40 章是所有 12 个模式的汇总图谱，可作为复习地图。

### 推荐阅读路径

**完整阅读**：按篇序阅读。第一篇（ch01-ch03）是必读地基，建立全局架构认知；第二篇（ch04-ch09）完整追踪一次交互生命周期；之后可以任意调整顺序。

**专题阅读**：
- 对工具系统感兴趣 → 直接从第三篇（ch10）开始
- 对多智能体感兴趣 → 先读第六篇（ch25），再读第七篇（ch29）
- 想快速获取可用模式 → 直接阅读第 40 章，按需跳回引用的章节
- 阅读第七篇（Swarm）前建议先读第六篇（Task 系统），因为 Swarm 架构依赖 Task 系统的概念

## 约定说明

**源码引用格式**：所有源码引用采用 `` `相对路径:行号` `` 格式，例如 `src/QueryEngine.ts:675`。引用前均已通过 `sed -n '[行号]p' [文件]` 验证，行号对应 Claude Code v2.1.88（重建版）。

**术语标注**：技术术语首次出现时标注英文原文，例如「死代码消除（Dead Code Elimination, DCE）」；后续章节直接使用简称「DCE」。完整术语表见**附录 F**。

**推断标注**：无直接源码依据的设计意图推断，会明确标注「（推断）」，区别于有源码证据的断言。

**章节交叉引用**：已在其他章节详细分析过的内容，用「详见第 X 章」代替重复讲解。

**代码注释翻译**：源码中的英文注释在书中统一翻译为中文，保留变量名和函数名原文；中文翻译后附英文原文，格式为「（原文："..."）」。

## 关于这个项目

本书基于 Claude Code v2.1.88 的重建源码分析写成。这份源码来自 npm 包意外保留的 source map 文件，是可合法获取的公开数据。书中分析的所有设计决策均以源码为证据，不依赖任何官方文档或内部信息。

书中提炼的 12 个工程模式是从源码中归纳的，不是 Anthropic 官方的架构术语——它们是这次追踪之旅的工程结晶，随时可以更名或重新分类。
